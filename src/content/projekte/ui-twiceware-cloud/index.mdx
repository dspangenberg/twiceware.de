Die Qualität von Software steigt und fällt mit ihrer Benutzeroberfläche (UI). So will das UI-Toolkit für die Entwicklung
einer Webanwendung gut gewählt sein.

Im vergangenen Jahr hat [shadcn/ui](https://ui.shadcn.com) die React-Community im Sturm erobert und einen [Paradigmenwechsel](https://apidog.com/de/blog/what-is-shadcn-ui-tutorial-de)
eingeleitet. In der Folge sind zahlreiche UI-Bibliotheken, die seiner Philosophie folgen, entstanden.

**twc-ui** ist das UI-Toolkit, das ich für die Entwicklung von internen und externen Projekten verwende.
Eine Sammlung von Komponenten, die in ihrem Aussehen, weitgehend der shadcn/ui-Designsprache folgen, aber für eine
bessere Zugänglichkeit (Barrierefreiheit) anstatt auf [Radix UI](https://www.radix-ui.com) auf den
[React-Aria-Komponenten](https://react-aria.adobe.com/) von Adobe aufbauen.

Bei der Entwicklung habe ich das DRY-Prinzip (Don’t repeat yourself) strikt umgesetzt, also versucht, redundanten Code
so weit wie möglich zu vermeiden. Insbesondere bei deren Einsatz, unterstützen die Komponenten einen DRY-eren
Einsatz als das Vorbild.

## Technologie-Stack
- React 19
- TailwindCSS
