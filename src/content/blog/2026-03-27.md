A CRM without emails isn’t a CRM. So how do we get our emails into the SaaS solution? Especially if we want to avoid implementing IMAP-based access and don’t want to reinvent the wheel with a bloated email client full of unnecessary emails and folders?

One solution is [Postal](https://docs.postalserver.io/). However, it has a drawback: the API doesn’t allow the creation of email addresses, so we have to work with wildcard addresses. As a result, we had to set up each tenant domain separately and send all emails blindly to the SaaS API, which would have led to a lot of unnecessary and potentially dangerous traffic.

So, the stage was set for the [Haraka Mail Server](https://haraka.github.io/); however, developing the appropriate plugin and configuring it was a bit tricky. Okay, we don't have an API for generating the emails too, but we have a simple INI file and can later use a CLI script.

```ini
[dropboxes]
invoice@example.com=https://tenant.your-dropbox-webhook.example.com/invoice@example.com/r0D4UZEf85Q34G6UbDfTFLmnpIhCT8fb
```
